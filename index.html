SQLite,
without
the SQL.

A lightweight Python wrapper around sqlite3 that speaks Python, not SQL. Create tables, insert rows, query data — all with clean method calls.

get started → documentation
pip install minidblite
example.py 
import minidblite

# one line to open (or create) a database
db = minidblite.create_database("shop.db")

# define your schema
db.new_table("products")
db.new_column("name",  str)
db.new_column("price", float)
db.new_column("stock", int)

# insert data with keyword arguments
db.add("products", name="Widget", price=9.99, stock=100)
db.add("products", name="Gadget", price=24.99, stock=50)

# pretty-print to terminal
db.get("products").decorate()
what it does

Everything you need.
Nothing you don't.

⚡
Zero SQL required
Create tables, insert, query, update, delete — all through clean Python method calls.
🔒
Injection-safe
All values use parameterized queries. All identifiers are validated with a strict regex.
📋
Beautiful output
Call .decorate() on any result to get a formatted table printed directly to your terminal.
📦
Structured responses
Every write operation returns {success, message, data} — no silent failures, no crashes.
🔄
Context manager
Use with create_database() as db to have the connection closed automatically.
📤
JSON & CSV export
Export any query result to JSON or CSV with a single method call.
quickstart

Up and running
in 30 seconds.

terminal 
# install
pip install minidblite
quickstart.py 
import minidblite

# ── 1. Open database ─────────────────────────────────
db = minidblite.create_database("app.db")

# ── 2. Define table ───────────────────────────────────
db.new_table("users")
db.new_column("name", str)
db.new_column("age",  int)
db.new_column("score", float)

# ── 3. Insert ─────────────────────────────────────────
db.add("users", name="Ali",  age=20, score=9.5)
db.add("users", name="Sara", age=25, score=8.2)

# ── 4. Query ──────────────────────────────────────────
db.get("users").decorate()          # all rows, pretty table
db.get("users", name="Ali")          # filtered

# ── 5. Update ─────────────────────────────────────────
db.update("users", row_id=1, name="Vali")

# ── 6. Delete ─────────────────────────────────────────
db.delete("users", name="Vali")       # first match
db.delete("users", is_all=True)       # all rows

# ── 7. Context manager (auto-close) ───────────────────
with minidblite.create_database(":memory:") as db:
    db.new_table("tmp")
    db.add("tmp", name="ephemeral")
api reference

Documentation0.1.0

FACTORY create_database(db_name="database_session.db") → Database
+
Creates or opens a SQLite database file and returns a connected Database object. Pass ":memory:" for a temporary in-memory database — useful for testing.
parameters
db_name str Path/filename for the database file. Defaults to "database_session.db" in the current directory.
returns → Database — a connected database instance
example
db = minidblite.create_database()                   # default file
db = minidblite.create_database("my_app.db")          # named file
db = minidblite.create_database(":memory:")           # in-memory, no file

# context manager — auto-closes on exit
with minidblite.create_database("app.db") as db:
    db.new_table("...")
METHOD close() → dict
+
Closes the database connection. Called automatically when using the context manager.
returns → dict — structured success response
QUERY list_tables() → list[str]
+
Returns a sorted list of all user-created table names in the database.
example
tables = db.list_tables()
# → ['products', 'users']
WRITE new_table(table_name: str) → dict
+
Creates a table with an auto-increment id primary key. Safe to call on an existing table — it's a no-op. Sets internal table context for subsequent new_column() calls.
parameters
table_name str Name of the table. Must match [A-Za-z_][A-Za-z0-9_]*
returns → dict — {success, message, data: {table}}
WRITE new_column(column_name, data_type, table_name=None) → dict
+
Adds a column to a table using ALTER TABLE. When table_name is omitted, uses the most recently created table. Silently skips if the column already exists.
parameters
column_name str Name for the new column.
data_type type Python type: str, int, float, bool, or bytes. See the Types tab.
table_name str | None Target table. Defaults to the last table created with new_table().
returns → dict — {success, message, data: {column, type, table}}
example
db.new_table("orders")
db.new_column("item",  str)         # uses "orders" context
db.new_column("qty",   int)
db.new_column("total", float)
db.new_column("note",  str, table_name="orders")  # explicit
QUERY schema(table_name: str) → dict
+
Returns column metadata for a table — name, SQL type, nullable, default value, and whether it's a primary key.
example
info = db.schema("users")
# info['data']['columns'] →
# [{'name': 'id', 'type': 'INTEGER', 'primary_key': True, ...},
#  {'name': 'name', 'type': 'TEXT', 'nullable': True, ...}]
WRITE drop_table(table_name: str) → dict
+
Permanently drops a table and all its data. Returns an error dict if the table doesn't exist.
WRITE add(table_name, **columns) → dict
+
Inserts a new row. Column values are passed as keyword arguments. Uses parameterized queries — SQL injection is not possible.
parameters
table_name str The table to insert into.
**columns Any One or more column=value pairs. At least one is required.
returns → dict — {success, message, data: {id: <new row id>}}
example
result = db.add("users", name="Ali", age=20)
# → {'success': True, 'data': {'id': 1}}

new_id = result["data"]["id"]
QUERY get(table_name, **filters) → QueryResult
+
Fetches rows from a table. Without filters, returns all rows. Filters are AND-joined. Returns a QueryResult object — never raises, returns empty result on error.
parameters
table_name str The table to query.
**filters Any Optional column=value conditions, AND-joined.
returns → QueryResult — iterable result object (see QueryResult tab)
example
db.get("users")                         # all rows
db.get("users", name="Ali")              # filtered
db.get("users", name="Ali", age=20)    # multiple filters (AND)
db.get("users").decorate()              # pretty-print
WRITE update(table_name, row_id: int, **new_values) → dict
+
Updates one or more columns on a specific row identified by its id. Returns an error dict if no row with that id exists.
parameters
table_name str The table containing the row.
row_id int The id of the row to update. Must be a positive integer.
**new_values Any Column=value pairs to apply. At least one required.
returns → dict — {success, message, data: {id, updated: {...}}}
example
db.update("users", row_id=1, name="Vali")
db.update("users", row_id=1, name="Vali", age=21)
WRITE delete(table_name, is_all=False, **filters) → dict
+
Deletes rows matching the given filters.
parameters
table_name str The table to delete from.
is_all bool False → delete only the first match (by lowest id). True → delete all matches.
**filters Any Optional column=value conditions. Without filters + is_all=True → deletes entire table.
returns → dict — {success, message, data: {deleted_count}}
example
db.delete("users", name="Ali")                    # first match
db.delete("users", is_all=True, name="Ali")         # all with name Ali
db.delete("users", is_all=True)                    # ⚠ truncate table
db.delete("users")                                  # → error (safe guard)

db.get() always returns a QueryResult object — never a raw list. It supports iteration, indexing, length, and several export methods.

QueryResult — all operations
result = db.get("users")

# pretty table in terminal
result.decorate()
result.decorate(tablefmt="grid")       # any tabulate format

# access data
result.records                          # [{'id':1, 'name':'Ali', ...}]
result.rows                             # [(1, 'Ali', 20), ...]
result.columns                          # ['id', 'name', 'age']
result[0]                              # first row as dict
result[0]["name"]                      # 'Ali'

# iteration
for row in result:
    print(row["name"], row["age"])

# export
json_str = result.to_json()
csv_str  = result.to_csv()

# checks
len(result)                             # 2
if result:                              # True if rows exist
    ...

Every write method returns a structured dict — never raises an unhandled exception. Check result["success"] to branch on success or failure.

✓ success
{
  "success": True,
  "message": "Row inserted ...",
  "data": { "id": 3 }
}
✗ failure
{
  "success": False,
  "message": "Table not found",
  "error":   "...",
  "data":    None
}
handling responses
result = db.add("users", name="Ali", age=20)

if result["success"]:
    print(f"Created row id=", result["data"]["id"])
else:
    print("Error:", result["message"])

Pass Python built-in types to new_column(). They are mapped to SQLite column types automatically.

db.new_column("text_col",  str)
db.new_column("int_col",   int)
db.new_column("float_col", float)
db.new_column("bool_col",  bool)
db.new_column("blob_col",  bytes)

db.add("example",
    text_col="hello",
    int_col=42,
    float_col=3.14,
    bool_col=True,
    blob_col=b"data"
